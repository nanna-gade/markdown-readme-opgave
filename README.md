# Markdown Dokumentation

## Fordele ved at bruge Markdown:
* Høj læsbarhed: Teksten forbliver læselig i sin rå form, da syntaksen er minimalistisk og intuitiv.

* Universel kompatibilitet: Filerne gemmes som .md (ren tekst), hvilket betyder, at de kan åbnes i alle teksteditorer på alle operativsystemer.

* Effektivitet: Formatering udføres direkte med tastaturet uden brug af mus eller grafiske menuer, hvilket bevarer skriverens fokus.

* Versionsstyring: Markdown er velegnet til systemer som Git, da ændringer kan spores præcist linje for linje.

* Nem konvertering: Dokumenter kan hurtigt eksporteres til HTML, PDF eller Word-formater via værktøjer som Pandoc.

* Fremtidssikring: Da formatet ikke er bundet til en specifik softwareproducent, vil filerne forblive læsbare uanset fremtidige programopdateringer.

## Eksempel på et array i JavaScript
```
const frugter = ["Æble", "Banan", "Appelsin"];

for (let i = 0; i < frugter.length; i++) {
  console.log(frugter[i]);
}
```

## Eksempel på en navigation med 3 links i HTML
```
<nav>
  <ul>
    <li><a href="index.html">Forside</a></li>
    <li><a href="om-os.html">Om os</a></li>
    <li><a href="kontakt.html">Kontakt</a></li>
  </ul>
</nav>
```
